임시적로 필요한 데이터를 메모리에 저장하기엔 비효율적입니다.

때문에 CPU 내부에 임시적으로 데이터를 저장할 수 있는 메모리가 있으며, 이곳이 바로 **레지스터**(Register)입니다.

주로 연산을 하기 위해 메모리에서 가져온 데이터나 데이터를 임시로 보관하거나, 주소를 저장하기 위해 쓰입니다.

데이터 임시 보관만 하는건 아닌데, 여러가지 레지스터의 종류에 대해 알아보겠습니다.

---

## 일반 레지스터

응용 프로그램에 의해 변경될 수 있는 레지스터이기 때문에, 사용자 가시 레지스터(User Visible Register)라고 불립니다.

### 데이터 레지스터 (DR, Data Register)

위에서 설명한대로 메모리에서 가져온 데이터를 임시적으로 저장할때 쓰이는 레지스터입니다.

아래의 주소 레지스터와 더불어 **일반 레지스터**라 부릅니다.

### 주소 레지스터 (AR, Address Register)

데이터나 명령어가 저장된 메모리의 주소를 임시적으로 저장하는 용도입니다.

후술할 MAR과의 차이점은, 주소 레지스터는 메모리 조작을 위한 레지스터입니다.

데이터 레지스터와 합쳐 **범용 레지스터**라고도 불립니다.

---

## 특수 레지스터

데이터 레지스터, 주소 레지스터 외에 다른 레지스터 또한 존재하는데, 이들은 응용 프로그램에서 접근할 수 없기 때문에, 사용자 불가시 레지스터(User Invisible Register)라고 불립니다.

### 프로그램 카운터 (PC, Program Counter)

[바이트 코드 등을 작성해봤다면](../making-a-tiny-bytecode-and-vm) 알겠지만, CPU가 다음 명령어를 실행하기 위해, 명령어의 행을 기억해뒀다가 제어장치에게 알려줍니다.

명렁어 포인터(Instruction Pointer)라고도 불립니다.

### 명령어 레지스터 (IR, Instruction Register)

현재 실행중인 명령어를 임시적으로 저장하는 용도입니다. 제어장치가 이 레지스터에 저장된 명령어를 해석하고 처리합니다.

### 메모리 주소 레지스터 (MAR, Memory Address Register)

CPU가 메모리에 접근하기 위해, 메모리 주소를 임시적으로 저장하는 레지스터입니다.

### 메모리 버퍼 레지스터 (MBR, Memory Buffer Register)

메모리에서 가져온 데이터나, 메모리로 이동할 데이터를 임시적으로 저장하는 레지스터입니다.

MBR은 MAR과 함께 작동합니다.

### 프로그램 상태 레지스터 (PSR, Program Status Register)

어떤 상태인지 등을 임시적으로 저장합니다.

PSR은 대부분 1바이트 또는 2바이트 이며, 각 비트에 해당되는 상태 여부가 존재합니다.

플래그 레지스터 (Flag Register) 등으로 불리기도 합니다.

### 누산기 레지스터 (AC, Accumulator Register)

산술 논리 장치(ALU, Arithmetic Logic Unit)에 존재하며, 연산을 위해 필요한 데이터를 메모리에서 가져온 후 AR에 임시로 저장합니다.

최종적으로 MBR을 통해 메모리로 이동됩니다.

---

이밖에도 IO 주소 레지스터, IO 버퍼 레지스터 등의 레지스터가 존재하지만, 이는 나중에 따로 설명하겠습니다.
